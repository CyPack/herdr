# Herdr Native-FM Errors

| Error | Cause | Fix | Date |
|-------|-------|-----|------|
| `codex mcp list` can expose credentials embedded in command arguments during an MCP audit | Codex masks values in the Env column but prints the Args column verbatim | Prefer structured TOML/plugin parsing that emits only server names, transports, enabled state, env key names, and path health; never surface raw `codex mcp list` output | 2026-07-14 |
| A narrow line-range read around an MCP section can still expose credentials | The requested range can spill into the adjacent `[mcp_servers.<name>.env]` table | Never use raw line windows for MCP config; parse TOML and emit a strict allowlist of safe fields | 2026-07-14 |
| `apply_patch` rejects a Herdr module edit with `Invalid Context` | The patch assumed a stale or reconstructed file header instead of the exact current worktree context | Inspect the narrow target region first, then reapply against the exact adjacent line; verify status to prove the rejected patch made no partial edit | 2026-07-14 |
| codebase-memory tools are listed but every call fails and port 9100 has no listener | The 26-server proxy cold start takes about 54 seconds, but readiness killed it after 45 seconds; cached tool schemas hid the dead transport | Without killing/restarting user processes, start a failed inactive backend if required, measure cold start on an isolated port, set readiness to a bounded 120 seconds inside a 150-second systemd budget, require exact 26/26 status plus real initialize/tools, then run the built-in MCP call and full reindex | 2026-07-14 |
| ShellCheck SC2016 flags the launcher prompt containing `$herdr-native-fm` | The skill trigger is intentionally literal inside single quotes and must not be expanded as a shell variable | Keep single quotes, add an adjacent explanation and a narrowly scoped `shellcheck disable=SC2016`, then rerun ShellCheck | 2026-07-14 |
| A multi-command verification shell continued to stage and commit after `cargo fmt --check` failed | The shell invocation did not enable fail-fast behavior, so newline-separated commands continued after a nonzero exit | Start multi-step gate/stage/commit commands with `set -euo pipefail`; immediately format, recheck, targeted-stage, and amend the unpublished commit before production work continues | 2026-07-14 |
| Linux `cargo clippy --all-targets -- -D warnings` rejects the bounded worker loop with `clippy::while-let-loop` | The worker used `loop` plus `let Some(request) = ... else { break; }`, which is correct but violates the repository's warning-as-error gate | Use the behavior-equivalent `while let Some(request) = take_next_request(...)` form, rerun the full B1 target set, then rerun Linux clippy before committing | 2026-07-14 |
| A scratch Herdr checkout pointed at the canonical repo target leaves link search paths into a deleted `/tmp/.../vendor/libghostty-vt/zig-out/lib` | Herdr's vendored build script records checkout-absolute paths in Cargo build output, so sharing target metadata across checkouts is unsafe | Give every scratch checkout its own target; if contaminated, confirm the stale path in `target/debug/build/herdr-*/output` and run `cargo clean -p herdr` before rebuilding the canonical checkout | 2026-07-14 |
| `cargo nextest list --message-format terse` is rejected before listing tests | This installed nextest supports `auto`, `human`, `oneline`, `json`, and `json-pretty`, not `terse` | Use `--message-format oneline`, inspect the selected inventory, and never count the rejected command as test evidence | 2026-07-14 |
| Kitty rejects `--allow-remote-control` while preparing an isolated host test | Kitty 0.46 exposes remote control as a config override rather than that CLI flag | Launch the throwaway host with `-o allow_remote_control=yes --listen-on unix:<throwaway-socket>`; use a unique X11 class/title and only semantic targeted input | 2026-07-14 |
| `cargo nextest list --ignored` is rejected as an unexpected argument | This installed nextest exposes ignored selection through `--run-ignored`, not a standalone `--ignored` flag | Use `cargo nextest list --locked --run-ignored only -E 'test(<name>)'` and never run a real-host ignored probe outside its isolated host contract | 2026-07-14 |
| `index_repository(mode="fast")` crashes the codebase-memory child with `munmap_chunk(): invalid pointer` and later MCP calls return an empty error | Native parallel Tree-sitter extraction corrupts memory while deleting parser/tree state; `mcp-proxy.service` remains active but the codebase-memory child core-dumps | Stop MCP retries and do not restart/kill the proxy; run the supported one-shot CLI with `CBM_WORKERS=1 codebase-memory-mcp cli index_repository '<json>'`, then prove freshness with CLI `index_status`, `search_graph`, and `get_code_snippet` against the same graph store | 2026-07-14 |
| A plugin manifest fixture fails with `invalid_plugin_min_herdr_version` before the action behavior under test is reached | Herdr requires every linked plugin manifest to declare a non-empty compatible `min_herdr_version`, even in focused unit fixtures | Include the current compatible baseline `min_herdr_version = "0.6.10"` in every valid plugin fixture; reserve omission or invalid values only for tests that explicitly target version validation | 2026-07-15 |
| A filesystem-preservation assertion expects the fixture filename bytes but reads `[120]` | The shared `TempDir::file(name)` helper writes the fixed payload `b"x"`; the test assumed it wrote `name` without inspecting the helper | Read the fixture helper before asserting payload bytes, or write the exact test payload explicitly; keep identity and content-preservation assertions separate | 2026-07-15 |
| A shell-based `rg` audit prints `command not found` for text inside backticks | Backticks inside a double-quoted shell command are command substitutions, even when intended as a literal Markdown search pattern | Put literal backtick-bearing regexes in single quotes or pass them as non-shell arguments; verify the failed read-only audit made no repository change before retrying | 2026-07-15 |
| A focused delete test expects a request but the controller emits none | Direct controller fixtures do not inherit cursor-only authority; destructive action authority requires an explicit current multi-selection, even for one path | Call `replace_selection(0)` in the fixture before requesting Delete, then assert the exact ordered path authority and keep cursor focus separate from selection authority | 2026-07-15 |
| A compound Nextest filter is rejected before running tests | Multiple nested filter terms were combined with unbalanced parentheses | Prefer one balanced regex expression such as `test(/(delete|file_operation)/)`; run `cargo nextest list -E` first when building a new complex filter and never count a parse failure as test evidence | 2026-07-15 |
| `Justfile` lookup fails even though the repository has Just recipes | The canonical recipe file is lowercase `justfile` on this checkout and the filesystem is case-sensitive | Inspect `justfile`; when `just` is absent, execute every command in the applicable recipe directly and record each gate separately | 2026-07-15 |
| A wrapped `exec_command` cell reports completion while its returned nested command still has a live `session_id` | The orchestration cell completed after yielding the nested PTY handle; it did not prove the Cargo process exited | Treat any returned `session_id` as authoritative, poll it with `write_stdin` until an explicit `exit_code`, and never launch another Cargo process while that handle is live | 2026-07-15 |

| A lifecycle test passes alone but intermittently misses fresh-process state under full-suite parallel load | The fixture mixes real `Instant::now()` wrappers with synthetic millisecond offsets, so a delayed real event can be newer than later synthetic evidence | Drive every event in the fixture through explicit `_at` seams using one monotonic synthetic clock, then require the exact test, related lifecycle family, and a fresh full suite | 2026-07-15 |
